◆ gps_data_parse()

GPS gps_data_parse ( const char * gps_src,
const unsigned short gps_src_len )

解析NMEA格式的GPS原始数据，提取各数据段信息

参数
gps_src输入的NMEA格式GPS原始数据字符串
gps_src_len输入数据的长度
返回
GPS 包含所有解析后GPS数据的结构体
注解
根据编译时定义的宏(ENABLE_GGA/ENABLE_GLL等)决定解析哪些NMEA数据段 未启用的数据段将返回默认值 支持解析的NMEA数据段包括：
  • GGA(全球定位系统固定数据)
  • GLL(地理定位信息)
  • GSA(当前卫星信息)
  • RMC(推荐最小特定GPS/TRANSIT数据)
  • VTG(地面速度信息)
  • GSV(可见卫星信息)
  • UTC(时间信息)

◆ gps_global_init()

void gps_global_init ( void )

初始化全局GPS数据结构

该函数用于初始化全局GPS数据结构体gps_all，根据编译时定义的宏(ENABLE_GGA/ENABLE_RMC等) 来初始化对应的NMEA语句解析数据结构。每个启用的NMEA语句类型都会被初始化为默认值。

注解
该函数应在系统启动时调用一次，用于初始化GPS解析模块的全局数据结构
当前实现不使用动态内存分配，所有缓冲区均为静态分配

支持的NMEA语句类型包括：

  • GGA: 全球定位系统定位数据
  • GLL: 地理定位信息
  • GSA: 当前卫星信息
  • RMC: 推荐最小定位信息
  • VTG: 地面速度信息
  • GSV: 可见卫星信息

每个语句类型的初始化值均为零或空字符串

◆ lat_dm_to_decimal()

double lat_dm_to_decimal ( double lat_dm,
char lat_dir )

将度分格式(DM)的纬度转换为十进制格式

参数
lat_dm度分格式的纬度值，格式为DDMM.MMMM（度+分）
lat_dir纬度方向，'N'表示北纬，'S'表示南纬
返回
double 转换后的十进制纬度值，范围为[-90.0, 90.0]
注解
转换公式：十进制纬度 = 度 + (分 + 分的小数部分)/60 南纬(S)返回负值，北纬(N)返回正值
