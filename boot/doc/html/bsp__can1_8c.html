◆ BSP_CAN1_Init()

void BSP_CAN1_Init ( void )

初始化CAN1控制器

完成CAN1控制器的完整硬件配置，包括：

  • 控制器初始化
  • 标准帧和扩展帧的收发邮箱配置
  • 接收过滤器设置（全通模式）
  • 中断回调函数注册
  • 中断优先级设置
  • 启动接收操作
注解
该函数应在系统初始化阶段调用，且只调用一次
使用前需确保CAN1时钟已使能
中断优先级设置为0x02（较高优先级）

◆ BSP_CAN1_Send_EXT()

uint8_t BSP_CAN1_Send_EXT ( uint32_t id,
uint8_t data[64],
uint8_t length )

通过CAN1发送扩展帧数据（支持CAN FD）

参数
id29位扩展帧标识符
data待发送的数据缓冲区（最大64字节）
length数据长度（1-64字节）
返回
uint8_t 发送状态码：
  • 0: 发送成功
  • 1: 发送失败
  • 2: 中止传输失败
注解
该函数采用阻塞式发送，最大重试500次后强制中止
警告
数据长度超过64字节将导致内存越界

通过CAN1总线发送标准帧数据（11位ID）

参数
id11位标准帧ID
data待发送的数据数组（固定8字节）
注解
该函数采用阻塞式发送，最多尝试500次后强制中止
如果发送超时，会先中止当前传输再重新尝试发送一次
警告
数据长度固定为8字节，超出部分将被忽略

◆ CAN1_Callback()

void CAN1_Callback ( uint32_t instance,
can_event_t event,
uint32_t buffIdx,
void * flexcanState )

CAN1中断回调函数，处理接收和发送完成事件

参数
instanceCAN实例号（未使用）
event触发的事件类型（CAN_EVENT_RX_COMPLETE或CAN_EVENT_TX_COMPLETE）
buffIdx缓冲区索引（未使用）
flexcanStateFlexCAN状态指针（未使用）
注解
此函数在中断上下文中执行，必须保持简短高效
接收到的标准帧和扩展帧消息会被分别处理
扩展帧消息会被转换为统一格式并通过消息队列发送给上层任务

主要功能：

  • 处理CAN接收完成事件
  • 读取标准帧和扩展帧消息
  • 转换扩展帧消息格式
  • 通过FreeRTOS队列发送消息指针
  • 清空接收缓冲区标志
