◆ BSP_UART1_Init()

void BSP_UART1_Init ( void )

初始化UART1外设

该函数完成UART1的硬件初始化配置，包括：

  1. 根据预定义配置初始化UART控制器
  2. 设置UART1中断优先级
  3. 启动首次接收操作
注解
该函数应在系统初始化阶段调用，且只调用一次
中断优先级设置为0x02（较高优先级）
使用轮询方式启动首次接收

◆ BSP_UART1_Printf()

void BSP_UART1_Printf ( char * fmt,
... )

通过UART1发送格式化字符串

该函数提供类似printf的格式化输出功能，通过UART1串口发送格式化后的字符串。 函数内部使用固定大小的缓冲区(100字节)存储格式化结果。

参数
fmt格式化字符串，遵循标准printf格式规范
...可变参数列表，与格式化字符串中的占位符对应
注解
函数会阻塞直到所有数据发送完成
警告
格式化后的字符串长度不能超过100字节，否则会导致缓冲区溢出

void UART1_Rx_Callback ( void * driverState,
uart_event_t event,
void * userData )

UART1接收中断回调函数

该函数在UART1接收到数据时被调用，处理接收到的数据并将其放入消息队列供上层任务处理。

参数
driverState驱动状态指针（未使用）
event触发回调的事件类型（UART_EVENT_RX_FULL表示接收完成）
userData用户自定义数据指针（未使用）
注解
该函数运行在中断上下文中，必须保持简短高效
使用双缓冲机制避免数据竞争：Uart1为硬件缓冲区，uart1_recv为应用缓冲区
每次只接收1字节数据，直到检测到换行符或缓冲区满
异常
无显式抛出异常，但内部会处理STATUS_BUSY等错误状态
